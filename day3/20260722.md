1. HTTP 상태코드(200, 400, 500번대)의 의미를 학습했다.
2. 400번대는 사용자(클라이언트) 요청 문제, 500번대는 서버 내부 문제라는 점이 인상적이었다.
3. 딕셔너리를 사용해 상태코드별, 시간대별, URL별, IP별 요청 수를 집계했다.
4. sorted()와 enumerate()를 활용해 집계 결과를 정렬하고 상위 데이터를 추출하는 방법을 익혔다.
5. 리스트와 딕셔너리를 함께 사용하면 데이터를 체계적으로 관리하고 원하는 정보를 쉽게 추출할 수 있다는 점을 배웠다.
6. 집계한 데이터를 JSON으로 저장하여 이후 그래프나 시각화에 활용할 수 있다는 흐름을 이해했다.
