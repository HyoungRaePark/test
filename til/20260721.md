## 오늘 배운 내용

### 1. Python 정규표현식을 활용한 로그 파싱

오늘은 `re` 모듈을 이용하여 로그 파일을 읽고 원하는 데이터를 추출하는 방법을 학습했다.

주요 학습 내용은 다음과 같다.

* `Path(__file__).parent`를 이용하여 현재 파일 기준으로 로그 파일 불러오기
* `re.compile()`을 사용하여 로그 형식에 맞는 정규표현식 작성
* `match.groupdict()`를 통해 매칭된 데이터를 딕셔너리 형태로 반환
* 함수(`parse_line`)를 만들어 코드 재사용성 향상
* 파싱 성공/실패 여부를 확인하며 로그 검증

특히 `groupdict()`를 사용하면 정규표현식에서 지정한 이름(`ip`, `datetime`, `method`, `status` 등)을 키로 하는 딕셔너리를 바로 얻을 수 있다는 점이 인상적이었다.

## 2. Git 기본 사용법

GitHub 저장소와 로컬 프로젝트를 연결하고 첫 커밋을 진행했다.

학습한 명령어는 다음과 같다.

```bash
git status
git add .
git commit -m "첫 커밋"
git remote add origin <repository_url>
git push -u origin main
```

처음에는 GitHub 계정 정보를 등록하기 위해

```bash
git config --global user.name
git config --global user.email
```

설정을 확인했고, 이후 원격 저장소를 연결하여 GitHub에 정상적으로 업로드했다.

## 3. Git 작업 흐름 이해

Git의 기본 작업 흐름을 이해했다.

```
파일 수정
    ↓
git status
    ↓
git add .
    ↓
git commit -m "변경 내용"
    ↓
git push
    ↓
GitHub 반영
```

또한 `git status`를 통해 현재 상태를 확인하는 습관이 중요하다는 점도 배웠다.

* `Changes not staged for commit`

  * 수정은 했지만 아직 `git add`를 하지 않은 상태

* `Changes to be committed`

  * `git add`가 완료되어 커밋 가능한 상태

* `nothing to commit, working tree clean`

  * 로컬 저장소와 Git 저장소가 동일한 상태
