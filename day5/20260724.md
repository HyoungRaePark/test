## TIL (2026-07-24)

- GitHub Actions를 이용해 프로젝트를 자동으로 배포하는 과정을 실습했다.
- 기존에는 로컬 환경(127.0.0.1)에서만 확인하던 웹페이지를 GitHub Pages를 통해 실제 URL에서 확인할 수 있었다.
- `.yml` 파일을 작성해 자동화 작업(Workflow)을 등록하는 방법을 배웠다.
- GitHub에 `push` 또는 `pull`을 수행하면 미리 작성한 Workflow가 실행되어 자동으로 빌드 및 배포되는 과정을 확인했다.
- 반복적으로 직접 배포하는 것이 아니라, 설정만 해두면 자동으로 실행된다는 점이 인상 깊었다.

### 느낀 점
GitHub Actions는 단순히 코드를 저장하는 기능이 아니라, 빌드와 배포까지 자동으로 처리할 수 있다는 점이 신기했다. `.yml` 파일 하나로 작업 과정을 정의할 수 있어 CI/CD의 기본적인 흐름을 이해할 수 있었고, 앞으로 프로젝트를 진행할 때 자동화의 중요성을 체감할 수 있을 것 같다.
